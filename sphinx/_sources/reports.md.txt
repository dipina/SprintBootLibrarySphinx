# Test & Quality Reports

All reports below are generated automatically by the CI/CD pipeline and published
alongside this Sphinx portal on GitHub Pages.

## JaCoCo — Code Coverage

JaCoCo instruments the bytecode and measures which lines, branches, and methods are
exercised by the test suite.

```{note}
**Direct link →** [JaCoCo Coverage Report](../site/jacoco/index.html)
```

The project enforces a minimum **25 % line coverage** via `mvn verify`.

## Surefire — Unit Test Report

The Surefire plugin records every test case result (pass / fail / skipped) and execution time.

```{note}
**Direct link →** [Surefire Unit Test Report](../site/surefire.html)
```

## Checkstyle — Code Style

Validates source code against Sun/Google coding conventions.

```{note}
**Direct link →** [Checkstyle Report](../site/checkstyle.html)
```

## PMD — Static Analysis

Detects common coding problems: unused variables, empty catch blocks, overly complex methods.

```{note}
**Direct link →** [PMD Report](../site/pmd.html)
**Direct link →** [CPD (Copy-Paste Detector)](../site/cpd.html)
```

## JDepend — Package Metrics

Measures coupling, cohesion, and instability at the package level.

```{note}
**Direct link →** [JDepend Report](../site/jdepend-report.html)
```

## Performance Test Report

Generated only when running the `performance` Maven profile. Shows throughput and
99th-percentile latency for annotated test scenarios.

```{note}
**Direct link →** [Performance Report](../reports/perf-report.html)
```

> Note: the performance report lives at `../reports/perf-report.html` (one level above
> `site/`), not inside it. This is because it is produced by a separate Maven profile
> and copied independently during the CI pipeline.

## Javadoc API Reference

Full Javadoc for all public classes and methods, generated by `maven-javadoc-plugin`
as part of `mvn site`. Requires the plugin to be declared in the `<reporting>` section
of `pom.xml` (see `javadoc_link` page).

```{note}
**Direct link →** [Javadoc](../site/apidocs/index.html)
```

## Doxygen Technical Reference

Doxygen parses the Java source and produces a cross-referenced HTML reference.
Richly documented classes include `BorrowingController`, `BookService`, and `Borrowing`.

```{note}
**Direct link →** [Doxygen HTML Reference](../doxygen/html/index.html)
```

## Generating All Reports Locally

```bash
# 1. Unit tests + JaCoCo
mvn test jacoco:report

# 2. Performance tests
mvn -Pperformance integration-test

# 3. Copy perf report into site directory
mvn -Pperformance resources:copy-resources@copy-perf-report

# 4. Full Maven site (Javadoc, Checkstyle, PMD, JDepend, Surefire, JaCoCo)
mvn site

# 5. Copy everything to docs/
mvn post-site
```

After step 4, open:

| Report | Local path |
|---|---|
| Javadoc | `target/site/apidocs/index.html` |
| JaCoCo | `target/site/jacoco/index.html` |
| Surefire | `target/site/surefire.html` |
| PMD | `target/site/pmd.html` |
| Checkstyle | `target/site/checkstyle.html` |
| JDepend | `target/site/jdepend-report.html` |
| Performance | `target/reports/perf-report.html` |
