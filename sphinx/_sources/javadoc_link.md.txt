# Javadoc API Documentation

Full Javadoc is generated by the **maven-javadoc-plugin** during `mvn site` and
published as part of the GitHub Pages deployment.

## Accessing the Javadoc

```{important}
**[→ Open Javadoc](../site/apidocs/index.html)**

*(Link works when browsing the published GitHub Pages site)*
```

## Required pom.xml configuration

Javadoc is only generated if `maven-javadoc-plugin` is declared in the `<reporting>`
section of `pom.xml`. Add the following block inside `<reporting><plugins>`:

```xml
<plugin>
  <groupId>org.apache.maven.plugins</groupId>
  <artifactId>maven-javadoc-plugin</artifactId>
  <version>3.6.3</version>
  <reportSets>
    <reportSet>
      <reports>
        <report>javadoc</report>
      </reports>
    </reportSet>
  </reportSets>
  <configuration>
    <!-- Suppress strict doclint errors that break the build -->
    <doclint>none</doclint>
  </configuration>
</plugin>
```

Without this, `mvn site` skips Javadoc entirely and `site/apidocs/` is never created.

## Generating Javadoc Locally

```bash
# As part of the full Maven site (recommended)
mvn site
# Open: target/site/apidocs/index.html

# Standalone
mvn javadoc:javadoc
# Open: target/reports/apidocs/index.html
```

## Relationship with Doxygen

| Tool | Output path | Strength |
|---|---|---|
| **Javadoc** | `site/apidocs/` | Standard Java API reference; IDE integration |
| **Doxygen** | `doxygen/html/` | Cross-referenced call graphs, class diagrams, richer navigation |
